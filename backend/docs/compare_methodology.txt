SECTION 1: WHAT IS A COMPARISON QUERY?
=============================================================================

A comparison query is any question where the user asks to:
- Compare two or more NEPSE stocks side by side
- Ask "which is better" between two symbols
- Ask to "compare X and Y"
- Ask about "X vs Y"
- Ask about multiple symbols in a single query

Examples:
  "compare NABIL and NICA"
  "which bank is better NABIL or EBL?"
  "tell me about NABIL and NICA and their latest news"
  "compare CHCL NHPC and HIDCL"

SECTION 2: MANDATORY RESPONSE STRUCTURE FOR COMPARE QUERIES
=============================================================================

Always present ALL requested symbols. Never omit a symbol. Follow this order:

STEP 1 — ONE-LINE SUMMARY
One sentence naming both stocks, their sector, and the key difference.
Example: "NABIL and NICA are both Commercial Banks; NABIL leads on EPS (35.18 vs 1.76)
and profitability, while NICA has higher trading volume."

STEP 2 — PRICE TABLE (Markdown)
Always present a side-by-side markdown table:

| Metric          | SYMBOL_A     | SYMBOL_B     |
|-----------------|-------------|-------------|
| LTP (NPR)       |             |             |
| Change (%)      |             |             |
| Volume          |             |             |
| Day Range       |             |             |
| 52W High        |             |             |
| 52W Low         |             |             |
| VWAP            |             |             |

STEP 3 — TECHNICAL INDICATOR TABLE
Show indicators for ALL symbols in one table. Show value + signal only.
Do NOT explain what each indicator means in a compare response.

| Indicator   | SYMBOL_A       | SYMBOL_B       |
|-------------|----------------|----------------|
| RSI (14)    |                |                |
| MACD        |                |                |
| EMA 20      |                |                |
| EMA 50      |                |                |
| BB Upper    |                |                |
| BB Lower    |                |                |
| BB Position |                |                |

STEP 4 — RELATIVE STRENGTH (2-3 sentences)
State which stock is technically stronger and why, using only the numbers shown.
Example: "NABIL's price is above both EMA-20 (525.6) and EMA-50 (510.0), suggesting
sustained upward momentum. NICA is below EMA-20 (367.9) with RSI at 51 (neutral).
NABIL shows stronger near-term momentum."

STEP 5 — SECTOR AND PEER CONTEXT
State the sector both belong to. Name 3-5 peer stocks.
Example: "Both NABIL and NICA are Commercial Banks (18 peers).
Key peers: EBL, SCB, GBIME, SBL, KBL."

STEP 6 — FUNDAMENTAL QUICK COMPARISON (if data available)
| Metric      | SYMBOL_A | SYMBOL_B |
|-------------|----------|----------|
| EPS (NPR)   |          |          |
| P/E Ratio   |          |          |
| ROE (%)     |          |          |
| NAV (NPR)   |          |          |
| NPM (%)     |          |          |

STEP 7 — NEWS PER SYMBOL
Present news under separate bold headers per symbol.
**SYMBOL_A — Recent News**
1. Headline — Source, Date
   One sentence context (from snippet only, no fabrication).

**SYMBOL_B — Recent News**
1. Headline — Source, Date

STEP 8 — DISCLAIMER (mandatory, always last)
"DISCLAIMER: This is for educational purposes only. Not financial advice."

SECTION 3: SIGNAL LABELS (use these exact labels — no variations)
=============================================================================

RSI Signal Labels:
  RSI below 30  → "Oversold"
  RSI 30 to 50  → "Neutral-Weak"
  RSI 50 to 70  → "Neutral"
  RSI above 70  → "Overbought"

MACD Signal Labels:
  Positive value → "Bullish"
  Negative value → "Bearish"

BB Position Labels:
  Price above BB Upper → "Above Upper Band"
  Price near upper half → "Near Upper Band"
  Price near midline   → "Mid Band"
  Price near lower half → "Near Lower Band"
  Price below BB Lower → "Below Lower Band"

EMA Trend Labels:
  Price > EMA-20 and EMA-20 > EMA-50 → "Bullish Alignment"
  Price < EMA-20 and EMA-20 < EMA-50 → "Bearish Alignment"
  Mixed positions                      → "Consolidating"

SECTION 4: RULES FOR COMPARE RESPONSES
=============================================================================

RULE 1: Always present data for EVERY symbol mentioned in the query.
If user asks "compare NABIL, NICA, and EBL", show all three symbols.

RULE 2: The SQL DATA block in context IS the price — read it directly.
Never say "unable to retrieve" if the data block exists.

RULE 3: Use tables for 2+ symbols. Prose only for single-symbol queries.
For 4+ symbols, use two separate tables (max 3 columns per table).

RULE 4: Do NOT explain what RSI, MACD, Bollinger Bands, EMA mean
in a compare response. Just show value and signal label.
Only explain if the user explicitly asks "what is RSI?" as a standalone question.

RULE 5: Do NOT give buy/sell recommendations. Present data only.
Acceptable: "NABIL shows stronger momentum than NICA."
Not acceptable: "You should buy NABIL over NICA."

RULE 6: For "which is better" queries, present data objectively.
End with: "Investors should consider their own risk tolerance and investment goals."

RULE 7: Never write "[context unclear]", "N/A" without reason, or placeholder text.
If data is truly missing, write "Not available in current data."

RULE 8: Always end with the DISCLAIMER line.

NABIL and NICA are both Commercial Banks listed on NEPSE.

| Metric     | NABIL   | NICA    |
|------------|---------|---------|
| LTP (NPR)  | 530.00  | 353.00  |
| Change (%) | -0.36%  | -2.71%  |

| 52W High   | 562.00  | 428.40  |
| 52W Low    | 471.00  | 304.50  |

| Indicator   | NABIL           | NICA            |
|-------------|-----------------|-----------------|
| RSI (14)    | 54.0 (Neutral)  | 51.0 (Neutral)  |
| MACD        | +3.09 (Bullish) | +6.99 (Bullish) |
| EMA 20      | 525.6           | 367.9           |
| BB Position | 52% (Mid Band)  | 19% (Near Lower)|

Relative Strength: NABIL's price (530) is above EMA-20 (525.6), indicating positive
momentum. NICA at 353 is below EMA-20 (367.9), suggesting near-term weakness. Both
show neutral RSI, so neither is overbought or oversold.

Sector: Both are Commercial Banks (18 peers: EBL, SCB, GBIME, SBL, KBL).

| Fundamental | NABIL  | NICA   |
|-------------|--------|--------|

| NAV (Rs)    | 235.64 | 198.33 |
| NPM (%)     | 44.56  | 4.51   |

NABIL is fundamentally the stronger bank with top EPS, ROE, and NPM in the sector.
NICA's high P/E of 190 reflects very low earnings — investors should monitor Q3 results.

SECTION 7: COMMON QUERY PATTERNS AND ROUTING
=============================================================================

Query Pattern → Expected Route:
"compare X and Y"              → ROUTE_COMPARE → sql + graph + news
"X vs Y"                       → ROUTE_COMPARE → sql + graph + news
"which is better X or Y?"      → ROUTE_COMPARE → sql + graph
"tell me about X and Y"        → ROUTE_FULL_AGENT → sql + graph + news + vector
"X and Y price"                → ROUTE_COMPARE → sql only
"compare X Y Z" (3+ symbols)   → ROUTE_COMPARE → sql + graph

For ROUTE_COMPARE: always fire sql_tool for ALL extracted symbols,
then graph_tool for sector/peer context, then news_tool concurrently.
